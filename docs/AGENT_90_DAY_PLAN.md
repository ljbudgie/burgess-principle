# Burgess Principle — 90-Day Agent Transformation Plan

> Companion to [`AGENTS.md`](../AGENTS.md). The rules that never change live there.
> The plan that ages quickly lives here so routine plan updates do not churn the
> file every coding agent loads first.

**Primary Goal:** Elevate this project to 10/10 groundbreaking status by executing
this 90-day plan in priority order, producing concrete deliverables.

Track live status against this plan in [`STATUS.md`](../STATUS.md).

## Phase 1 — Immediate Polish & Accessibility (Week 1)

- Rewrite and dramatically improve the root `README.md` to make the "why this
  matters now" hook irresistible for developers, lawyers, policymakers, and
  disabled users.
- Create a new `GETTING_STARTED.md` with 5-minute quickstart, interactive
  decision tree, and "copy-paste" templates for the top 5 use cases (energy
  bills, benefits, lending, content moderation, reasonable adjustments).
- Add a live "Burgess Score" badge generator and a simple web demo (or
  embeddable widget) that lets anyone test the binary on a fictional or real
  decision.

## Phase 2 — Evidence & Academic / Regulatory Credibility (Weeks 2–4)

- Research and draft a concise 8–12 page whitepaper (*"The Burgess Test: A
  Binary Standard for Meaningful Human Involvement under UK GDPR and the EU AI
  Act"*) suitable for SSRN, arXiv, or submission to journals.
- Produce 3–4 new high-quality case studies (one in healthcare / medical AI, one
  in financial services, one in public-sector benefits, one in platform content
  moderation) with clear before / after NULL → SOVEREIGN outcomes and measurable
  user benefit.
- Generate a **Responses to Critiques** living document that proactively
  addresses the top 8 objections with calm, evidence-based rebuttals.
- Map the principle explicitly against EU AI Act Article 14, NIST AI RMF, and
  the latest UK ICO guidance on automated decision-making.

## Phase 3 — Ecosystem & Adoption Acceleration (Weeks 3–6)

- Create ready-to-merge integration modules or PR templates for the
  highest-impact projects: LangChain / LangGraph, LlamaIndex, AutoGen, CrewAI,
  OpenClaw, Hermes-agent, and at least two major UK public-sector or
  disability-tech repos.
- Draft and (where possible) open PRs or issues in 5–7 high-visibility
  repositories that would benefit from Burgess governance.
- Build a new `adoption/` directory with: institutional adoption tracker,
  "Burgess-Ready" certification checklist for organisations, and a public
  leaderboard of compliant vs non-compliant systems.

## Phase 4 — Community, OSS Friendliness & Global Reach (Weeks 4–8)

- Propose and implement a clarified, contributor-friendly licensing FAQ and
  `CONTRIBUTING.md` that lowers barriers for open-source projects while
  protecting the certification mark.
- Create a `translations/` folder with professional-quality versions of the core
  principle + one-pager in at least French, German, Spanish, and Simplified
  Chinese.
- Prepare the full USPTO certification mark filing package (based on the
  existing UK application).
- Design a lightweight **Burgess Ambassadors** programme with tiered recognition
  for contributors, early adopters, and case winners.

## Phase 5 — Technical Maturity & Tooling (Ongoing, parallel)

- Audit and harden the Sovereign Personal Vault (add formal security review
  checklist, test vectors, and a third-party audit template).
- Extend the Python toolkit with a `burgess-audit` CLI command that
  organisations can run against their decision logs.
- Add optional "multi-human" and "escalation" modes to the binary test while
  keeping the core single-question purity.
- Create an Iris v2 roadmap that turns the companion into a fully local-first,
  privacy-first **Sovereign Rights Assistant**.

## Phase 6 — Narrative & Virality (Throughout)

- Expand `LINEAGE.md` into a standalone, beautifully formatted `ORIGIN_STORY.md`
  (or short book) that can be shared independently.
- Script and storyboard 3 short explainers (60–90 seconds each): one for
  developers, one for lawyers / policymakers, one for disabled users and
  advocates.
- Identify and draft 10 high-signal social / X / LinkedIn posts, GitHub
  discussions, and newsletter angles that leverage the medieval civic root +
  modern AI accountability angle.

## First deliverable to produce on a fresh agent run

An updated root `README.md` + `GETTING_STARTED.md` that makes a first-time
visitor immediately understand why this is the missing piece in 2026 AI
governance.

Read first: the current root `README.md`, `LINEAGE.md`, `PRINCIPLE.md`, all
papers in `/papers/`, `ADOPTION.md`, and the latest `CHANGELOG.md`.
